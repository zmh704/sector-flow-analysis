# A股板块资金流向分析系统

一个基于浏览器的前端分析页面，用于可视化展示 A 股行业板块与概念板块的资金流向数据，并辅以「今日推荐」「关注板块」「趋势对比」等深度分析能力。前端为原生 JavaScript + Chart.js，后端为极简的 Node.js 静态/接口服务器，无前端框架、无构建步骤。

## 功能特点

### 核心可视化
- **双视图展示**：行业板块与概念板块分开显示
- **横向柱状图**：Y 轴为板块名称，X 轴为主力净额（亿元），叠加前一日期数据（虚线）便于对比
- **直观配色**：红色表示净流入（正值），绿色表示净流出（负值）；连续两日净流入的板块名称加粗高亮
- **Top-3 数据标签**：柱条末端自动标注数值

### 深度分析
- **🔥 今日推荐**：基于多条件筛选的龙头股票推荐（连续流入天数、所属板块关注状态、成交额放量/缩量、涨跌幅限制等）
- **📌 关注板块**：满足净额为正、连续流入、成交额趋势等条件的板块，自动建立行业↔概念共同股票配对
- **趋势对比弹窗**：板块近 10 日主力净额/成交额趋势图，右侧面板支持「涉及股票 / 今日推荐 / 关注板块」三个页签
- **个股详情**：弹窗内嵌入新浪图表（分时/日K/周K/月K）或 TradingView 行情，并可一键跳转东方财富完整行情页
- **股票预选**：可将股票标记为预选，状态持久化到 `localStorage`

### 数据管理
- **多日期数据**：自动扫描 `data/` 目录下所有 `*板块资金流向*.json` 文件，按修改时间排序生成日期切换按钮
- **Excel 上传解析**：上传 `.xlsx` 源数据，服务端实时解析生成 JSON 并入库
- **批量重处理**：`reprocess.js` 批量重解析 `data/源数据/` 下的 Excel，覆盖生成 JSON
- **静态列表回退**：本地服务器不可用时回退到 `list.json`（适用于 GitHub Pages 部署）

### 交互
- **键盘快捷键**：`←/→` 切换日期，`1-9` 快速选择日期，`Esc` 关闭弹窗
- **搜索过滤**：查看全部弹窗支持按板块名称 / 涉及股票名称模糊搜索
- **表格排序**：所有表格支持点击表头排序

## 快速开始

### 方式一：本地服务器（推荐，支持 Excel 解析）

1. 安装 [Node.js](https://nodejs.org/)（建议 16+）
2. 安装依赖：
   ```bash
   npm install
   ```
3. 启动服务器：
   ```bash
   npm start
   ```
   或双击 `start.cmd`（Windows）
4. 浏览器访问 <http://localhost:3000>

### 方式二：静态部署（GitHub Pages，仅查看）

1. 执行 `node generate-list.js` 生成 `list.json`
2. 将 JSON 数据文件放入 `data/` 目录后再次生成 `list.json`
3. 部署到任意静态托管服务（如 GitHub Pages），页面会自动回退到读取 `list.json`

## 数据准备

### 数据文件位置

所有板块资金流向 JSON 文件统一放在 `data/` 目录下，文件名需匹配 `*板块资金流向*.json`。

### 从 Excel 生成数据

**方式 A — 页面上传**：点击页面右上角「📊 解析数据」，选择 `.xlsx` 文件，服务端解析后自动写入 `data/`。

**方式 B — 批量重处理**：将 Excel 放入 `data/源数据/`，文件名需含日期（如 `6月24日.xlsx`），执行：
```bash
node reprocess.js
```

## 使用说明

### 顶部操作栏
- **日期按钮**：横向滚动列表，点击切换当前展示日期；当前日期高亮
- **📊 解析数据**：上传 Excel 文件并解析

### 主图表区
- **流入 / 流出**单选：独立控制行业/概念图表显示净流入或净流出板块
- **查看全部**：打开数据表格弹窗，支持搜索、排序、过滤无效板块，点击行进入板块详情

### 🔥 今日推荐 / 📌 关注板块
- 点击今日推荐股票标签 → 打开该股票所属（最优）板块的趋势弹窗
- 点击关注板块标签 → 打开该板块的趋势弹窗
- 连续流入天数 ≥ 3 天的板块/股票以红色高亮

### 趋势对比弹窗
- **左侧**：板块详情（近 10 日趋势图） / 个股详情（新浪图表或 TradingView）
- **右侧**：涉及股票 / 今日推荐 / 关注板块 三个页签
- **标题**：点击回到当前板块视图
- **相关板块**：展示与当前板块有共同股票的关联板块，点击可切换

### 键盘快捷键
| 按键 | 功能 |
|------|------|
| `←` / `→` | 切换到前一个 / 后一个日期 |
| `1` - `9` | 快速选择第 N 个日期 |
| `Esc` | 关闭当前弹窗 |

## 服务端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/list` | 返回 `data/` 下匹配 `*板块资金流向*.json` 的文件列表（按修改时间排序） |
| POST | `/api/parse` | 接收 `multipart/form-data` 上传的 Excel，解析后写入 `data/`，返回 `{ success, industries, concepts, file }` |

其余路径按静态文件服务（已做路径遍历防护）。

## 技术栈

- **Chart.js 4.4.1**：主图表与趋势图渲染（本地引入）
- **TradingView tv.js**：个股行情嵌入（CDN）
- **新浪财经图表**：个股分时/K 线图片
- **xlsx**：服务端 Excel 解析
- **原生 JavaScript（ES6+）**：无前端框架、无构建步骤
- **CSS Grid & Flexbox**：响应式布局

## 可配置项

核心阈值常量集中在 `config.js` 顶部，便于调优：

| 常量 | 默认 | 说明 |
|------|------|------|
| `FOCUS_MIN_DAYS` | 1 | 关注板块连续流入最低天数 |
| `HIGHLIGHT_MIN_DAYS` | 3 | 板块标签红色高亮阈值 |
| `LEADER_STOCK_MIN_DAYS` | 1 | 今日推荐股票连续流入最低天数 |
| `LEADER_GAP` | 1 | 股票天数 vs 所属板块最大天数容差 |
| `VOLUME_WINDOW` | 5 | 成交量比较窗口 |
| `RATIO_TURNOVER_LOW` | 0.9 | 成交额缩量阈值 |
| `RATIO_TURNOVER_HIGH` | 1.5 | 成交额放量阈值 |
| `CHANGE_LIMIT_PCT` | 5 | 放量时涨跌幅限制（%） |
| `TREND_CHART_DAYS` | 10 | 趋势图显示天数 |
| `STOCK_CHART_SOURCE` | `sina_chart` | 个股图表默认数据源（`sina_chart` / `tradingview`） |

今日推荐的筛选条件在 `leaders.js` 的 `passesLeaderConditions()` 中，各条件独立成方法，可通过注释开关。

## 常见问题

**Q: 页面提示「本地服务器和静态列表均不可用」**
A: 请先用 `npm install && npm start` 启动本地服务器；若要静态部署，先执行 `node generate-list.js` 生成 `list.json`。

**Q: 图表不显示**
A: 打开浏览器控制台查看错误。确认 `chart.umd.min.js` 可访问、`data/` 下有匹配的 JSON 文件。

**Q: 数据显示「NaN」**
A: JSON 数据格式有误，检查数值字段（`主力净额`、`成交额`）是否为数字类型。

**Q: Excel 解析失败**
A: 检查 Excel 是否包含必要列（股票简称、行业板块、概念板块、主力净额、成交额），列名识别规则见 `analyze.js` 的 `detectColumns()`。

**Q: 多日期数据不合并**
A: 本系统不合并多日期数据，而是按日期分别加载、切换查看。文件名需匹配 `*板块资金流向*.json`。
